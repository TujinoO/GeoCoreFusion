# RGB–NIR–SWIR 配准优化与人工复核

## 为什么旧版结果不可用

旧版流程只在全幅低分辨率代理图上执行一次 ECC，并把 NIR、SWIR 分别配准到 RGB。该设计存在三个直接问题：

1. 全幅预览将 2048 像素 RGB 和 320 像素 HSI 压到同一尺寸，局部尺度误差在预览上不明显，但会在 1024 像素融合 ROI 中放大成几十个 RGB 像素。
2. 旧版条带漂移由低置信度相位相关直接插值得到，可能用局部相关性换取不真实的弯曲。
3. NIR、SWIR 没有利用 1050–1450 nm 重叠波段彼此约束，因此即使各自与 RGB 的代理分数较高，两套高光谱数据仍可能错位。

## v2 配准链路

新版采用两级、三重约束：

1. **全幅粗定位**：保留受物理尺度限制的 ECC，只用于寻找共同覆盖范围和 ROI。
2. **真实 ROI 精配准**：在最终进入融合的低分辨率观测网格上，使用梯度、边缘邻近距离和 Laplacian 构造跨模态结构特征，再估计受尺度、剪切、平移范围限制的局部仿射变换。
3. **NIR–SWIR 联合约束**：用两台高光谱传感器的 1050、1150、1250、1350、1450 nm 重叠波段估计附加变换。只有重叠波段分数达到最小增益、且不会明显损害 NIR–RGB 对齐时才接受。
4. **安全门控**：ROI 配准状态参与总质量状态。全幅代理图得分不能再掩盖最终融合网格的失败。

逐行非刚性修正代码仍保留为实验选项，但默认关闭。真实数据测试表明，未经严格地质控制点约束的逐行修正容易过拟合模态差异；当前正式结果使用更稳健的 ROI 仿射和 NIR–SWIR 联合约束。

## 配准审查模式

完整融合前先运行：

```powershell
$env:PYTHONPATH='E:\Code\GeoCoreFusion\src'
python -m geocorefusion.cli register configs\3dssz_roi_registration_v2.yaml
python -m geocorefusion.cli register configs\zkh3_roi_registration_v2.yaml
```

该命令只生成小型审查文件，不生成 2 GB 级融合立方体。重点查看：

- `registration_nir_roi_after.png`：RGB–NIR 最终 ROI 边缘叠加；
- `registration_swir_roi_after.png`：RGB–SWIR 最终 ROI 边缘叠加；
- `registration_nir_swir_overlap_after.png`：NIR–SWIR 重叠波段叠加；
- `registration_nir_roi_checkerboard.png`、`registration_swir_roi_checkerboard.png`：检查托盘边框、槽线和主要岩块轮廓是否连续；
- `registration_review.json`：精配准前后分数、矩阵、门控结论和所有预览路径。

红绿叠加图中，跨模态反射率差异本身会产生彩色纹理，不能只凭颜色多少判断。人工验收应优先检查托盘外框、纵向隔板、箱体横边以及大型岩块几何轮廓是否连续，并结合棋盘图判断。

## v3：分段几何与 RGB 空间细节

v3 进一步解决两类问题：

1. 3DSSZ 中外框已对齐、中央隔板仍错位，说明残差不是一个全局仿射变换，而是横向非线性镜头/推扫畸变。代码从跨越大部分扫描行的纵向隔板响应中提取高置信度峰值，对 NIR、SWIR 建立共享的单调分段横向校正，并在 RGB 几何改善与 NIR–SWIR 重叠波段一致性之间自动选择校正强度。
2. v2 只让 RGB 引导低秩系数边界，不能恢复高光谱均匀区域中的细颗粒纹理。v3 从 RGB 亮度提取多尺度高频残差，生成所有高光谱波段共用的乘性增益图。共用增益不会给不同波段注入不同纹理；其低分辨率退化响应被迭代约束到 1 附近，因此新增信息主要位于原始 HSI 观测的空间高频近零空间。

必须区分三类产品：

- `rgb_reference.png` 是原始 RGB 参考影像；
- `fused_false_color_2200_1650_900.png` 是融合高光谱立方体的 2200/1650/900 nm 假彩色，不是普通 RGB 彩色照片；
- `spatial_detail_before_after.png` 并排显示 RGB、未注入高频细节的 HSI 基础结果和最终 RGB 细节增强 HSI。

RGB 没有被解释成 691 nm 以下的连续光谱，也没有作为三个新增光谱波段拼入立方体。它提供空间网格、材料边界和全波段共享的高频增益；NIR/SWIR 决定 691–2518 nm 的光谱形状与低频辐射信息。

正式 v3 配置和结果目录：

- `configs/3dssz_roi_fusion_v3.yaml` → `runs/3dssz_roi_fusion_v3`；
- `configs/zkh3_roi_fusion_v3.yaml` → `runs/zkh3_roi_fusion_v3`。
